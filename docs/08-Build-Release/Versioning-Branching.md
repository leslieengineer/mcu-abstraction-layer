# Versioning & Branching Strategy

## 1. Semantic Versioning

```
MAJOR.MINOR.PATCH[-prerelease]
```

| Component | Increment When | PX4 Example |
|-----------|---------------|-------------|
| **MAJOR** | Breaking API change, new flight stack architecture | `2.0.0` |
| **MINOR** | New feature release cycle (yearly cadence) | `1.16.0` |
| **PATCH** | Bug fix, no new features | `1.16.1` |
| **Pre-release** | Release candidate builds | `1.16.0-rc1`, `1.16.0-beta1` |

## 2. Branch Strategy

```
main ─────────────────────────────────────────▶  (development, always latest)
  │
  ├── release/1.15 ─────────────────▶  (LTS: 1.15.0, 1.15.1, 1.15.2)
  │
  ├── release/1.16 ─────────────────▶  (current: 1.16.0, 1.16.1)
  │
  ├── pr-branch / feature─* ──▶ (merge → main via PR)
  │
  └── hotfix/* ──▶ (cherry-pick → release/1.16 + main)
```

| Branch | Purpose | Merge Target | Naming |
|--------|---------|-------------|--------|
| `main` | Active development (latest) | — | `main` |
| `release/1.16` | Stable release branch | Cherry-picks from `main` | `release/X.Y` |
| Feature PR | New features | `main` (via GitHub PR) | Free-form (PR-based) |
| `hotfix/*` | Critical production fix | `release/X.Y` + `main` | `hotfix/short-description` |

## 3. Tag Convention

```
vMAJOR.MINOR.PATCH[-rcN]
```

- Tag on `release/X.Y` branch after release gate approval
- Annotated tags with release summary
- Examples: `v1.16.0`, `v1.16.1`, `v1.16.0-rc1`, `v1.16.0-beta1`

## 4. Firmware Version Embedding

PX4 auto-generates version from Git tags:

```c
// Auto-generated by CMake from git describe
#define PX4_GIT_VERSION_STR   "v1.16.1-42-g1a2b3c4"
#define PX4_GIT_TAG_STR       "v1.16.1"
#define PX4_GIT_BRANCH_NAME   "main"
#define PX4_BUILD_URI         "px4_fmu-v6x_default"
```

- Version visible in MAVLink `AUTOPILOT_VERSION` message
- Version visible in NSH console: `ver all`
- Version stored in ULog file header for every flight

## 5. Board Configuration Files

PX4 uses `.px4board` Kconfig files for per-board feature selection:

```
boards/px4/fmu-v6x/default.px4board
boards/px4/fmu-v6x/bootloader.px4board
boards/px4/fmu-v6x/debug.px4board
```

These define which drivers, modules, and options are compiled for each board variant.
